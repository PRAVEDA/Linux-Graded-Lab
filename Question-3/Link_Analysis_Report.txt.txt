Question 3: File System and Link Analysis

============================================================

1. Creating the Original File

Command:
echo "Linux Link Experiment" > original.txt

Observation:
A file named original.txt was created successfully. It will be used to demonstrate the behavior of hard links and symbolic links.

============================================================

2. Creating a Hard Link

Command:
ln original.txt hardlink.txt

Observation:
A hard link named hardlink.txt was created. It shares the same inode as the original file.

============================================================

3. Creating a Symbolic Link

Command:
ln -s original.txt symlink.txt

Observation:
A symbolic link named symlink.txt was created. It stores the pathname of the original file.

============================================================

4. Inode Numbers

Command:
ls -li

Output:
(Paste your ls -li output here.)

Observation:
The original file and hard link have the same inode number, indicating that they reference the same file. The symbolic link has a different inode because it is a separate file containing the path to the original.

============================================================

5. Metadata Information

============================================================

6. Effect of Deleting the Original File

Command:
rm original.txt

Observation:
The original file was deleted successfully.

Command:
cat hardlink.txt

Output:
Linux Link Experiment

Observation:
The hard link continued to work because it references the same inode where the file data is stored.

Command:
cat symlink.txt

Output:
cat: symlink.txt: No such file or directory

Observation:
The symbolic link became broken because it points to the original file path, which no longer exists.

============================================================

Comparison Between Hard Link and Symbolic Link

Hard Link
-----------
• Shares the same inode as the original file.
• Remains functional even if the original file is deleted.
• Cannot normally link directories.
• Cannot span different file systems.

Symbolic Link
--------------
• Has a different inode.
• Stores the pathname of the original file.
• Becomes broken when the original file is deleted.
• Can link files across different file systems.

============================================================

Conclusion

This experiment demonstrated the difference between hard links and symbolic links. Hard links reference the same inode as the original file and remain accessible after deleting the original file. Symbolic links only store the file path and become invalid when the original file is removed. Linux utilities such as ls -li and stat help analyze inode information and file metadata effectively.

============================================================

Summary of Findings

• Original file and hard link shared the same inode.
• Symbolic link had a different inode.
• Hard link remained functional after deleting the original file.
• Symbolic link became broken after deleting the original file.
• Metadata was successfully collected using the stat command.
